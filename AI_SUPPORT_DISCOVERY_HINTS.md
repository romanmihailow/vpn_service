# AI Support — UX-подсказки (discovery hints)

**Дата:** 2025-03-15  
**Цель:** улучшить discoverability поддержки: пользователь понимает, что боту можно писать вопросы текстом.

---

## 1. Изменённые файлы

| Файл | Изменения |
|------|-----------|
| `app/messages.py` | Добавлены константы: `SUPPORT_DISCOVERY_TEXT`, `SUPPORT_AFTER_CONFIG_HINT`, `SUPPORT_AFTER_ACTIVATION_HINT` |
| `app/bot.py` | Импорт `SUPPORT_AFTER_CONFIG_HINT`; третье сообщение при отправке конфига: текст = `CONNECTION_INSTRUCTION_SHORT` + подсказка |
| `app/tg_bot_runner.py` | Импорт `SUPPORT_DISCOVERY_TEXT`; в `/start` к приветственным сообщениям дописывается подсказка |

---

## 2. Куда встроены подсказки

### 2.1. Приветственное сообщение /start

- **Где:** обработчик `cmd_start` в `app/tg_bot_runner.py`.
- **Когда показывается:**
  1. При успешной регистрации по реферальной ссылке: после `REF_LINK_WELCOME_TEXT` добавлен блок `SUPPORT_DISCOVERY_TEXT`.
  2. При «уже есть реферер» + активная подписка: то же — `REF_LINK_WELCOME_TEXT` + `SUPPORT_DISCOVERY_TEXT`.
  3. При обычном `/start` (без параметра или без успешной реф-регистрации): после `START_TEXT` добавлен блок `SUPPORT_DISCOVERY_TEXT`.
- **Кнопки и логика:** без изменений; клавиатуры (REF_TRIAL_KEYBOARD, get_start_keyboard) и реферальный flow не трогались.

### 2.2. После отправки конфига

- **Где:** `send_vpn_config_to_user` в `app/bot.py`.
- **Что менялось:** третье сообщение (короткая инструкция + кнопка «Нужна помощь») теперь отправляется с текстом  
  `CONNECTION_INSTRUCTION_SHORT + "\n\n" + SUPPORT_AFTER_CONFIG_HINT`.
- **Когда показывается:** при любой выдаче конфига (триал, оплата, промо, resend и т.д.) — везде, где вызывается `send_vpn_config_to_user`.

### 2.3. После успешной активации подписки / выдачи доступа

- Отдельного блока «после активации» в коде нет: сценарий «получил доступ» в текущей логике завершается отправкой конфига через `send_vpn_config_to_user`, поэтому подсказка **после конфига** (`SUPPORT_AFTER_CONFIG_HINT`) одновременно играет роль подсказки после активации.
- Константа `SUPPORT_AFTER_ACTIVATION_HINT` вынесена в `messages.py` для возможного использования в других местах (например, в webhook’ах при отдельном текстовом сообщении об успехе).

## 3. Тексты подсказок

### SUPPORT_DISCOVERY_TEXT (в /start)

```
💬 Если возникнут вопросы — просто напишите мне сообщением.

Например:
— VPN не работает
— вышли конфиг
— статус подписки
— как подключиться
```

### SUPPORT_AFTER_CONFIG_HINT (после конфига)

```
Если что-то не работает — просто напишите:
«VPN не работает»
или
«как подключиться»
```

### SUPPORT_AFTER_ACTIVATION_HINT (зарезервировано в messages.py)

```
Если возникнут сложности — просто напишите мне:
— VPN не работает
— вышли конфиг
— статус подписки
```

## 4. Подтверждение: бизнес-логика не изменена

- **Referral / onboarding:** не менялись; добавлено только присоединение строки к уже выводимому тексту.
- **Payment / webhook:** не менялись; подсказка добавляется только в общую функцию отправки конфига `send_vpn_config_to_user`.
- **FSM, support router, intents, guardrails, actions:** не трогались.
- **WireGuard, платежи, рефералы, админка:** без изменений.
- **Новые константы** используются только при формировании текста сообщений; условия вызова хендлеров и сценариев не менялись.

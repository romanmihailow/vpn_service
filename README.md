                 VPN

MaxNet VPN Bot__

Telegram-бот для автоматической выдачи и управления VPN-доступом на базе WireGuard.

Проект реализует полный цикл управления доступом:

выдача VPN-конфигураций;

контроль срока действия подписок;

автоматическое отключение доступа по окончании срока;

административное управление через Telegram.

Возможности
Для пользователей

Получение VPN-конфигурации WireGuard через Telegram-бота

Подключение через QR-код или .conf-файл

Просмотр статуса текущей подписки

Получение уведомлений о событиях, связанных с доступом

Для администраторов

Управление подписками через Telegram

Активация и деактивация VPN-доступа

Просмотр текущих и прошлых подключений

Ручная выдача или отзыв доступа при необходимости

Архитектура

Проект построен по модульному принципу:

Telegram Bot — интерфейс взаимодействия с пользователями

PostgreSQL — хранение подписок и технического состояния

WireGuard — VPN-туннели и ключевая инфраструктура

Фоновые задачи — контроль срока действия подписок

Основные компоненты
app/
├── tg_bot_runner.py        # точка входа Telegram-бота
├── bot.py                  # отправка конфигов, QR-коды, сообщения
├── db.py                   # работа с базой данных
├── wg.py                   # управление WireGuard (ключи, peer’ы)
├── config.py               # конфигурация проекта
├── logger.py               # логирование
├── yookassa_webhook_runner.py
├── heleket_webhook_runner.py

Хранение данных

Сервис обрабатывает только минимально необходимые данные, такие как:

Telegram ID пользователя

статус и срок действия доступа

технические параметры VPN-подключения

Проект не предназначен для хранения чувствительных персональных данных.

Автоматическое управление доступом

Фоновый воркер периодически:

проверяет подписки с истёкшим сроком действия;

деактивирует доступ в базе данных;

удаляет соответствующие peer’ы из WireGuard.

Развёртывание
Переменные окружения

Проект использует .env-файл для конфигурации окружения:

TELEGRAM_BOT_TOKEN=your_bot_token
ADMIN_TELEGRAM_ID=123456789

DB_HOST=localhost
DB_PORT=5432
DB_NAME=vpn_service
DB_USER=vpn_user
DB_PASSWORD=secret

WG_INTERFACE_NAME=wg0
WG_CLIENT_NETWORK_CIDR=32


Полный список параметров см. в app/config.py.

Запуск локально
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python -m app.tg_bot_runner

Запуск в Docker
docker compose build
docker compose up -d


Контейнер использует network_mode: host для взаимодействия с WireGuard-интерфейсом хоста.

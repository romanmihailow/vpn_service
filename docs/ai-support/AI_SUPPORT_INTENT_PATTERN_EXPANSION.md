# AI Support — расширение паттернов интентов

**Дата:** 2025-03-15  
**Цель:** улучшить распознавание интентов по естественным фразам пользователя без изменения архитектуры, guardrails, actions или router.

---

## Проблема (manual testing)

- **«проверь подписку»** → ожидался `subscription_status`, фактически fallback.
- **«помоги с подключением»** → ожидался `connect_help`, фактически fallback.

---

## Изменения в `app/support/intents.py`

Изменены только списки паттернов `CONNECT_HELP_PATTERNS` и `STATUS_PATTERNS`. Порядок проверки интентов в `classify_intent()` не менялся.

---

## 1. subscription_status (STATUS_PATTERNS)

**Было:**
```python
STATUS_PATTERNS = [
    r"до какого", r"до какой даты", r"когда истекает", r"срок подписки",
    r"статус подписки", r"активна ли подписка", r"подписка активна",
]
```

**Стало:**
```python
STATUS_PATTERNS = [
    r"до какого", r"до какой даты", r"когда истекает", r"срок подписк",
    r"статус подписк", r"активна ли подписк", r"подписк активна",
    r"проверь подписк", r"моя подписк", r"какая подписк",
    r"подписк работает",
]
```

**Что сделано:**
- В существующих паттернах подстрока **«подписк»** вместо «подписки»/«подписка», чтобы ловить варианты: подписку, подписки, подписка.
- Добавлены паттерны для фраз из тестов и типичных запросов:
  - **проверь подписк** — «проверь подписку», «проверь подписки»
  - **моя подписк** — «моя подписка», «моя подписку»
  - **какая подписк** — «какая подписка», «какая у меня подписка»
  - **подписк работает** — «подписка работает», «подписка ещё работает»

**Примеры фраз, которые теперь дают subscription_status:**
- проверь подписку  
- моя подписка  
- какая подписка  
- подписка активна  
- подписка работает  
- срок подписки / статус подписки / активна ли подписка (как и раньше)

## 2. connect_help (CONNECT_HELP_PATTERNS)

**Было:**
```python
CONNECT_HELP_PATTERNS = [
    r"как подключить", r"как установить", r"wireguard", r"настроить",
    r"импорт", r"qr", r"подключиться", r"не работает подключение",
]
```

**Стало:**
```python
CONNECT_HELP_PATTERNS = [
    r"как подключ", r"как установить", r"wireguard", r"настроить",
    r"импорт", r"qr", r"подключ", r"не работает подключ",
    r"помоги подключ", r"помоги с подключ", r"как настроить vpn",
    r"как подключить vpn", r"не могу подключ",
]
```

**Что сделано:**
- В существующих паттернах подстрока **«подключ»** вместо «подключить»/«подключиться»/«подключение», чтобы ловить: подключить, подключиться, подключение, подключению и т.д.
- Добавлены паттерны:
  - **помоги подключ** — «помоги подключить», «помогите подключиться»
  - **помоги с подключ** — «помоги с подключением», «помогите с подключением»
  - **как настроить vpn** — «как настроить vpn», «как настроить VPN»
  - **как подключить vpn** — «как подключить vpn»
  - **не могу подключ** — «не могу подключить», «не могу подключиться»

**Примеры фраз, которые теперь дают connect_help:**
- помоги подключить  
- помоги с подключением  
- как настроить vpn  
- как подключить vpn  
- не могу подключить  
- как подключиться / не работает подключение (как и раньше, плюс варианты с «подключ»)

# Статистика AI-support: handoff и fallback в /support_stats

## Что добавлено

В отчёт команды `/support_stats` добавлены два блока:

1. **Проблемные интенты (по handoff)** — топ-5 интентов по доле передач в живую поддержку (handoff).
2. **Проблемные интенты (по fallback)** — топ-5 интентов по доле ответов через безопасный fallback.

Цель — показывать не только объём по интентам, но и **где бот реально не справляется** (перевод в человека или универсальный ответ).

## Откуда берутся данные

- **total_count по интенту** — существующий запрос `get_support_conversation_intent_stats(hours=24)` к таблице `support_conversations`: количество записей с данным `detected_intent` за последние 24 часа.
- **handoff_count по интенту** — новый запрос `get_support_conversation_handoff_by_intent(hours=24)` к таблице `support_conversations`: количество записей с `handoff_to_human = TRUE` и данным `detected_intent` за последние 24 часа. Структура БД и остальные запросы не менялись.
- **fallback_count по интенту** — парсинг файла `support_ai.log` за последние 24 часа: подсчёт строк с `intent=<intent>` и `fallback=True`. В БД поле fallback не хранится, поэтому используется только лог.

## Как считается handoff_rate

Для каждого интента с `total_count > 0`:

- `handoff_rate = round(handoff_count / total_count * 100)` (проценты).
- Интенты сортируются по убыванию `handoff_rate`.
- В отчёт выводятся первые 5 строк в формате:  
  `• <русская_подпись_интента>: <total_count> / handoff <handoff_count> (<handoff_rate>%)`.

## Как считается fallback_rate

Для каждого интента с `total_count > 0`:

- `fallback_rate = round(fallback_count / total_count * 100)` (проценты).
- Интенты сортируются по убыванию `fallback_rate`.
- В отчёт выводятся первые 5 строк в формате:  
  `• <русская_подпись_интента>: <total_count> / fallback <fallback_count> (<fallback_rate>%)`.

Для подписей интентов используется существующий словарь `SUPPORT_STATS_INTENT_LABELS` (русские названия).

## Пример обновлённого отчёта

```
Статистика AI-поддержки (за 24 часа)

По источнику ответа:
rule: 41
memory: 8
faq_match: 2
openai: 21
fallback: 0

Популярные интенты:
не распознано: 34
реферальная программа: 13
помощь с подключением: 13
статус подписки: 11

Диагностика VPN:
VPN подключён (есть handshake): 3

Проблемные интенты (по handoff):
• не распознано: 34 / handoff 21 (62%)
• помощь с подключением: 13 / handoff 5 (38%)

Проблемные интенты (по fallback):
• не распознано: 34 / fallback 21 (62%)
```

Если по handoff или fallback нет ни одной записи за период, в соответствующем блоке выводится «(нет)».

## Ограничения

- Логика AI-support, FSM, платежи, CRM и обработка сообщений не менялись.
- Существующие SQL и структура таблиц не менялись (добавлен только новый запрос по уже существующей колонке `handoff_to_human`).
- Новые таблицы и колонки в БД не добавлялись.
- Fallback считается только по логу; при отсутствии или ротации лога блок «Проблемные интенты (по fallback)» может быть пустым или неполным.

# Аудит унификации пост-подключенческих сообщений (POST_VPN_CONNECTED)

**Дата:** 2025-03-12  
**Цель:** Оценить текущую реализацию unified post-connection messaging после внедрения `get_post_vpn_message(context)`: насколько удачно унифицированы сообщения, нет ли потери контекста, нужна ли доработка.

**Ограничение:** Только анализ, без изменений кода.

---

## 1. Все текущие места использования

Текущая реализация использует **три контекста**: `initial`, `success`, `followup`. Общий блок (тариф 270₽, экономия 10%, CTA /buy) — один (`POST_VPN_CONNECTED_COMMON`).

|----------|------------|---------|----------|---------------|------------|-----------------|
| **Первый handshake** | `tg_bot_runner.py` ~6976, job `auto_new_handshake_admin_notification` | Авто: обнаружен первый handshake по подписке (триал/промо) | `initial` | «VPN подключён 👍», стабильно, пробный 7 дней + COMMON | `HANDSHAKE_USER_CONNECTED_KEYBOARD`: [💎 Закрепить доступ — 270 ₽], [📅 Все тарифы] | `HANDSHAKE_USER_CONNECTED_TEXT` (тот же смысл, одна строка) |

Дополнительно по сценариям, которые **не** используют unified message:

- **Short confirmation (~60 сек):** `HANDSHAKE_SHORT_CONFIRMATION_TEXT` — без продажи, только «всё ок» + поддержка.
- **10m follow-up (само сообщение):** `HANDSHAKE_FOLLOWUP_10M_TEXT` — «Всё открывается нормально?» + кнопки [Всё работает], [Поддержка]; при нажатии «Всё работает» уже даётся unified `success`.
- **3d referral nudge:** `HANDSHAKE_REFERRAL_NUDGE_3D_TEXT` — отдельный текст и реферальная клавиатура; не unified.

| **success (checkpoint)** | Явно подтвердил «Да, всё работает» после вопроса бота. | Да. «Отлично, рады что всё работает» уместно. | Да по рефералу: раньше сразу шёл реферальный блок и кнопка «Пригласить друга»; сейчас только оплата. |

Вывод по контексту:

- **Где единый текст уместен:** первый handshake (initial), явный success по кнопкам (checkpoint, 10m), follow-up 2h.
- **Где может звучать неестественно или терять контекст:** config_check_now (вместо короткого «подключён» — длинный upsell); 24h (тот же текст, что 2h, без кнопок).
- **Где пользователь на другом этапе воронки:** 24h — уже получал 2h напоминание; повтор того же текста без нового CTA может снижать вовлечение.

## 3. Message density и повторение

Приблизительный таймлайн для пользователя с триалом:

| Момент | Что получает |
|--------|----------------|
| T+0 | Конфиг + инструкция + кнопка «Проверить подключение» |
| T+0 (при handshake) | **initial** — «VPN подключён», пробный 7 дней, тариф 270₽, [Закрепить], [Все тарифы] |
| T+~1 мин | Short confirmation — «Всё открывается нормально?» + [Поддержка], без продажи |
| T+~3 мин | Checkpoint «Удалось подключиться?» → при «Да» — **success** (тот же COMMON, другой интро) |
| T+10 мин | 10m — «Всё открывается нормально?» + [Всё работает], [Поддержка] → при «Всё работает» — **success** (третий раз тот же COMMON) |
| T+2h | **followup** — «Если VPN работает стабильно», COMMON, [Закрепить], [Пригласить друга], [Поддержка] |
| T+24h | **followup** — тот же текст, без кнопок |

Наблюдения:

- **Повторение нормальное:** initial и один раз success (checkpoint или 10m) — разный интро, один COMMON, ожидаемо.
- **Риск повторения:** один и тот же COMMON (тариф 270₽, экономия 10%, /buy) пользователь может увидеть: (1) при первом handshake, (2) при «Да, всё работает», (3) при «Всё работает» в 10m, (4) при 2h, (5) при 24h. До 5 раз за сутки — ощущение «бот повторяется» возможно.
- **Где может раздражать:** 24h тем же текстом, что и 2h, но без кнопок — и по содержанию, и по действиям выглядит как дубль.

## 4. Реферальная часть

**Раньше:**

- После «Да, всё работает» (checkpoint): второе сообщение — `REFERRAL_PROMPT_AFTER_CONNECTION_SUCCESS` и кнопка [👥 Пригласить друга]. Реферальный nudge был явным и сразу после успеха.
- После «Всё работает» (10m): клавиатура с [Купить подписку] и [Пригласить друга] — реферал рядом с оплатой.

**Сейчас:**

- После checkpoint и после 10m «Всё работает» везде одна клавиатура: `HANDSHAKE_USER_CONNECTED_KEYBOARD` — только [Закрепить доступ — 270 ₽] и [Все тарифы]. Кнопки «Пригласить друга» нет.
- Реферальный nudge после явного success (checkpoint и 10m) **исчез** в этих точках.
- В 2h по-прежнему есть [🤝 Пригласить друга] в `HANDSHAKE_FOLLOWUP_2H_KEYBOARD`.
- В 3d — отдельное реферальное сообщение и своя клавиатура.

**Влияние на referral funnel:** в двух ранних, высокоинтентивных точках (только что подтвердил успех) реферальный призыв и кнопка убраны; акцент смещён на оплату. Реферал остаётся только в 2h и 3d. Возможная потеря конверсии в реферальную программу на этапе «только подключился и сказал, что всё ок».

---

## 5. CTA consistency

- **Одна и та же клавиатура для initial + всех success:** `HANDSHAKE_USER_CONNECTED_KEYBOARD` (Закрепить 270₽, Все тарифы) — да, везде одна и та же.
- **Расхождения:**
  - **2h:** своя клавиатура — [Закрепить доступ], [Пригласить друга], [Поддержка]. Текст тот же (followup), но CTA шире (реферал + поддержка).
  - **24h:** тот же текст (followup), но **нет клавиатуры**. Пользователь не получает ни оплаты, ни реферала, ни поддержки кнопками — при том же тексте, что и в 2h. Это расхождение и по смыслу (повтор), и по действиям (нет CTA).
- **Раньше 2h/24h:** оба имели текстовый акцент «закрепить доступ», 2h мог иметь кнопки. Сейчас акцент сохранён, но 24h без кнопок при том же тексте — хуже по консистентности.

Итог: единый текст для initial/success и один тип клавиатуры для них — плюс. Для followup: 2h с расширенной клавиатурой — ок; 24h без клавиатуры при том же followup-тексте — минус.

---

## 6. Текстовая корректность

- **«Пробный доступ активен на 7 дней»:** есть только в интро `initial`. В `success` и `followup` этой фразы нет — корректно по времени (в 2h/24h не переутверждаем «7 дней»).
- **«Чтобы VPN не отключился после теста, можно закрепить доступ уже сейчас» (в COMMON):** по смыслу подходит и для initial, и для success, и для followup. Для 24h «уже сейчас» чуть менее срочно, но допустимо.
- **Конфликты с таймингом:** явных смысловых конфликтов нет. Слабое место — 24h: тот же блок, что и 2h, без нового повода или другого акцента (например, «если ещё не закрепили» или отдельный короткий вариант).

## 7. Финальный вывод

### 1. Unified approach overall: **MIXED**

- Плюсы: один общий блок (тариф, скидка, /buy), три осмысленных интро (initial / success / followup), единая оплатная клавиатура в точках «только подключился» и «всё работает».
- Минусы: ослаблен реферальный nudge в двух ранних точках; 24h без клавиатуры при том же тексте, что 2h; config_check_now превращён из короткого статуса в длинный upsell; риск ощущения повторения одного и того же сообщения до 5 раз за сутки.

### 2. Где унификация точно удалась

- Первый handshake (initial): контекст и текст совпадают.
- Единый COMMON и один тип клавиатуры для «только что подключился» и «подтвердил успех» (checkpoint, 10m) — предсказуемо и консистентно.
- Follow-up 2h: текст и расширенная клавиатура (оплата + реферал + поддержка) согласованы.

### 3. Где, возможно, потерян контекст

- **Checkpoint и 10m «Всё работает»:** потеря явного реферального сообщения/кнопки после success.
- **config_check_now при handshake:** потеря короткого «VPN уже подключён»; вместо него — полный upsell.
- **24h:** тот же текст, что 2h, но без кнопок — повтор без усиления CTA.

### 4. Рекомендация

**B. Добавить 2–3 контекстные вариации (и точечные правки), не откатывая унификацию:**

- В точках **success** (checkpoint и/или 10m) — вернуть реферальный элемент: либо вторую кнопку [Пригласить друга] в клавиатуре, либо короткий реферальный абзац/одну строку после unified message, без возврата к двум разным большим сообщениям.
- Для **config_check_now** при handshake — рассмотреть короткий ответ («VPN уже подключён ✅») с одной кнопкой типа [Закрепить доступ] или оставить один короткий абзац + ссылку на /buy, чтобы не дублировать полный COMMON в третий раз подряд.
- Для **24h:** либо другой короткий текст («Напоминаем: можно закрепить доступ по /buy»), либо ту же клавиатуру, что у 2h, чтобы не повторять один и тот же длинный текст без действий.

Унификацию initial + success (с доработкой реферала) и общий COMMON сохранить; точечно скорректировать config_check_now и 24h.

## 8. Итог

Post-connection message unification audit complete.  
**Verdict: MIXED** — унификация по смыслу и одному источнику текста достигнута и в целом уместна; для полного успеха стоит вернуть реферальный nudge в точках success и уменьшить повторение/непоследовательность в config_check_now и 24h (2–3 контекстные вариации или точечные правки без отката всей унификации).
